Crossplatform Multiplayer WebXR Rover

This is the NASA Perseverance rover built to WebXR through Unity and WebGL. The experience uses MRTK for crossplatform interactions, Photon Pun 2 for networked interactions and is hosted with github pages. This project has been developed by the NASA GRC GVIS lab in support of the EPFD project. This test was performed in the Boise State University Stein Luminary CAVE which is running a desktop instance of the WebXR Rover demo on the walls, while the Quest 2 and HoloLens 2 also interact with the same virtual rover through the web. Supported Devices: HoloLens 1&2 (Native browser support), Quest 1&2 (Native browser support), PCVR (WebXR API Emulator Plugin), Desktop (Native browser support), iOS (Mozilla WebXR Viewer because Safari doesn't support WebXR) and Android (Native browser support)

HoloLens 2 Low-Light Solution

This is a brief video showcasing the BSU NASA SUITS team's solution to low light scenarios with the HoloLens 2 which requires light to spatially map the environment. Our team 3D printed a mount for a powerful 1500 lumen light allowing us to provide sufficient light to enable spatial mapping functionality in unpredictable environments. This was done for our ARSIS 5.0 software as part of the NASA SUITS 2022 challenge (Spacesuit User Interface Technologies for Students) in which our software solution was tested in a dark environment to mimic the lighting on a lunar EVA.

Dynamic Realworld Environments in Unity with Cesium

In this project I setup a demo of Cesium for Unity deployed to the Quest 2. The Quest 2 references the Cesium API on start up and populates the environmental mesh and textures based on satellite data sets for that location. In the beginning I showcase the real-world location in google maps and then open up Cesium for Unity on the Quest 2 and pull in the data for that same location. If you pay close attention you'll notice the same buildings, and even the same vehicles parked on the side of the road between the google maps street view and Cesium's Environmental Mesh in Unity.

Virtual School Bus

Developed for Invent For the Planet 2019 Our challenge was to conceptualize and prototype an application of Virtual Reality to benefit the community within a 48 hr time period. Ultimately we settled on a Virtual Reality language learning game that would strive to fully immerse the player in the world of the language they are trying to learn helping them form stronger mental connections with what they are saying/hearing. We used Oculus OVR plugin with Unity to create a VR recreation of a Japanese city that the player needs to traverse by speaking Japanese with NPC's. This forces the player to speak conversationally and think more reactivley as opposed to following scripted conversations and responses. The ultimate goal of this is a more efficient and effective learning experience. Ultimately we didn't get the language translation to be fully functional due to lack of time, but we got a simple prototype working and managed to take second place in the competition.
